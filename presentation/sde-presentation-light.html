The framework in three ideas

Three pillars

Pillar 1

Eval validity is the product

An eval is trusted only when it is GREEN-on-right ∧ RED-on-wrong — it passes on the intended code and is proven to FAIL on a plausible mutant.

Pillar 2

Progressive rigor

One framework, three tiers: Tier 0 convention → Tier 1 gated skills → Tier 2 autonomous factory. You never install machinery you don’t need yet.

Pillar 3

Proof by dogfooding

It is built using itself. The proof is runnable; resumability is demonstrated by a cold agent recovering state from disk alone.

Why prose + bindings

Intent, then evidence

Each intent is a single checkable statement. Each binds an eval that must catch a violation of it. Coverage is derived: every intent → ≥1 eval, or the slice can’t advance.

Gates: required blocks · flag surfaces · advisory always allows.

Anti-collusion

The information barrier

The mutant author SEES
  • The intent (plain prose)
  • The code under test

…and must write a plausible diff that violates the intent.

The mutant author NEVER sees
  • The eval it must beat

Blind to the assertion, it can’t craft a strawman tuned to be caught. In Tier 2 this is an eval-blind git worktree.

Gold standard: a surgical mutant that breaks one intent facet while every other eval stays green — proof the eval is load-bearing and non-redundant.

Coverage validity

The bounded intent-audit

An independent auditor sees only the intent + the eval set — never the code — and must exhibit a betrayer: an implementation that passes every eval yet violates a named facet.
It answers the question trust alone can’t: is anything intended but un-evaluated?
Betrayer found → required new eval
2 clean rounds → satisfied
3-round cap → escalate, never spin

State model

Every fact is PRIMITIVE or DERIVED

Primitive — authored once

Written, never recomputed

objective · spec · spec.lock · tasks · trust.log · runs.log · decisions. Append-only where it counts, so a torn write is the worst damage.

Derived — pure f(disk)

Computed at read time, never stored

phase · coverage · eval-trust · the next action · the handoff dossier. Nothing to corrupt, nothing to go stale, no dual source of truth.

This is what makes “the next action is a pure function of disk” literally true — and why a fresh agent always agrees with the last one.

Tier 2 · the factory

A mechanical loop, fresh context per rung

framework/bin/sde-factory.shwhile sde next; do rc=$? # 0 done · 2 work · 1 error [ $rc = 0 ] && escalate done-when # never auto-succeed before=$(git rev-parse HEAD) claude -p "do ONLY the next action; commit; stop" [ HEAD = before ] && escalate no-progress done
Executor in the harness

Agents propose; the driver decides

Only the driver runs evals/mutants and writes the logs — a builder can’t grade its own work.

No session rots

One rung = one fresh claude -p

Nothing lives long enough to decay; the loop is the durable part, not the session.

Tier 2 · safety

Bounded autonomy — designed to halt

Hard stops

Caps everywhere

HEAD-based no-progress retry cap + a single MAX_STEPS budget + a .sde/STOP kill-switch + a steer file.

Never weaken a gate

Escalate, don’t cheat

A gate that won’t pass after N tries blocks + escalates. The loop never lowers a bar to get green.

Structural guards

Cheating can’t pay

Weakening an eval re-opens its trust; forged verdicts can’t (driver writes them); a vacuous eval can never reach done.

The driver commits to a branch and never pushes or deploys. Humans own “done.”
